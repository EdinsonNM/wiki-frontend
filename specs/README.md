# Specs

Usa esta carpeta para features medianas o grandes.

## Estructura

```text
specs/
  <feature-id>/
    spec.md
    plan.md
    tasks.md
```

## Cuando Crear Una Spec

- Hay mas de una pantalla, estado o flujo.
- Hay integracion con API o datos.
- Participan varios agentes.
- El cambio puede romper comportamiento existente.
- El trabajo tomara mas de una sesion.

Para bugs o tareas pequenas, usa Mini Spec en el issue, PR o conversacion.
