# Implement Feature

Implementa la feature aprobada.

Antes de editar:

- Confirma que existe spec, Mini Spec o task directa.
- Lee `AGENTS.md`.
- Lee solo documentos relacionados con la feature.
- Revisa `git status` si el proyecto usa git.

Reglas:

- Lee la spec o Mini Spec primero.
- Mantente dentro del ownership asignado.
- No reestructures areas fuera del scope.
- Reutiliza componentes, hooks y services existentes.
- Agrega tests proporcionales.
- Ejecuta validaciones relevantes.
- Actualiza docs si cambia comportamiento, comandos o arquitectura.

Reporta archivos modificados, comandos ejecutados y riesgos pendientes.
