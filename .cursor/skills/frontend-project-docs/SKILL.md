# Frontend Project Docs

## Read First

- `AGENTS.md`
- `docs/README.md`
- `.agents/skills/project-docs/SKILL.md`
- archivos modificados reales

## Workflow

1. Identificar que cambio ocurrio realmente.
2. Actualizar `docs/` segun el area afectada.
3. Registrar decisiones en `docs/architecture/decisions.md`, `docs/features/<feature>/decisions.md` o `docs/log/decisions-log.md`.
4. Documentar comandos solo si existen o fueron ejecutados.
5. Marcar riesgos pendientes sin inventar resultados.

## Do Not

- No escribir en `wiki/` salvo que cambie una regla reutilizable.
- No inventar comandos.
