# Solution Planning

## Read First

- `AGENTS.md`
- `wiki/index.md`
- `wiki/delivery/feature-solution-plan.md`
- spec o issue relacionado, si existe

## Workflow

1. Clasifica la tarea: feature, bug, refactor, QA, docs o release.
2. Elige modo: Spec Kit, Mini Spec o task directa.
3. Define comportamiento esperado y criterio de salida.
4. Lista archivos probables y validacion.
5. Indica si docs deben actualizarse.

## Output

```md
## Plan

### Modo
### Justificacion
### Comportamiento Esperado
### Criterio De Salida
### Archivos Probables
### Validacion
### Docs Afectadas
```
