# Project Docs

## Read First

- `AGENTS.md`
- `docs/README.md`
- documentos afectados
- archivos modificados reales

## Workflow

1. Identifica que cambio realmente ocurrio.
2. Actualiza docs del proyecto, no la wiki metodologica, salvo regla reusable.
3. Documenta comandos solo si existen o fueron verificados.
4. Marca pendientes cuando algo no se pudo validar.

## Do Not

- No inventes scripts.
- No conviertas `wiki/` en bitacora del proyecto.
