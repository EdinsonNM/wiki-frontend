# Frontend Spec Kit

## Read First

- `AGENTS.md`
- `wiki/delivery/spec-driven-development.md`
- `wiki/delivery/feature-solution-plan.md`
- `specs/README.md`
- `specs/_template/*`

## Workflow

1. Confirmar que la tarea requiere Spec Kit completo.
2. Crear o actualizar `specs/<feature-id>/spec.md`.
3. Crear o actualizar `specs/<feature-id>/plan.md`.
4. Crear o actualizar `specs/<feature-id>/tasks.md`.
5. Incluir estrategia de skills y subagentes si participaran varios agentes.
6. No implementar hasta que existan criterios de aceptacion y tasks claras.

## Do Not

- No crear specs para cambios triviales.
- No convertir la spec en documentacion decorativa.
- No implementar codigo dentro de este skill.
