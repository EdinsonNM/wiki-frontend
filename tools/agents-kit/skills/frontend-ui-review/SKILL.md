# Frontend UI Review

## Read First

- `AGENTS.md`
- `wiki/architecture/frontend-clean-architecture/presentation-design-system.md`
- `docs/design-system/overview.md`
- `docs/design-system/components.md`
- feature/spec relacionada

## Workflow

1. Revisar si existen componentes reutilizables.
2. Verificar estados loading, empty, error, success y disabled si aplican.
3. Revisar responsive basico.
4. Revisar accesibilidad basica: labels, focus, roles, contraste cuando aplique.
5. Reportar gaps concretos y archivos afectados.

## Do Not

- No redisenar producto fuera del scope.
- No crear componentes duplicados.
- No cambiar logica de negocio.
