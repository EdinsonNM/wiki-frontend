# Design System Diagrams

Guardar mapas visuales de componentes, tokens o migracion de design system.

## Usos Recomendados

- Mapa de componentes `ui`, `shared` y `domain`.
- Relacion entre tokens, themes y primitives.
- Plan de migracion desde componentes duplicados.
- Estados visuales de componentes complejos.

## Regla

Crear diagramas solo si reducen ambiguedad. Para inventarios simples, usar `docs/design-system/components.md`.
