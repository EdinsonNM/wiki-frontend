# Design System Migration

Usar cuando el proyecto existente tenga componentes duplicados, estilos dispersos o tokens inconsistentes.

## Flujo

1. Inventariar componentes existentes.
2. Detectar duplicaciones.
3. Elegir una pantalla o feature piloto.
4. Migrar sin cambiar comportamiento observable.
5. Agregar o preservar tests.
6. Documentar equivalencias y pendientes.

## Salida Esperada

```md
## Migracion Visual

### Inventario
### Duplicaciones
### Piloto
### Cambios Permitidos
### Fuera De Alcance
### Validacion
```
