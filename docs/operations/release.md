# Release

## Checklist

- Tests relevantes ejecutados.
- Build ejecutado.
- Docs actualizadas.
- Riesgos pendientes documentados.

## Evidencia De Entrega

```md
## Release YYYY-MM-DD

### Cambios
### Comandos Ejecutados
### Resultado
### URLs / Artefactos
### Riesgos Pendientes
```

## Reglas

- No ocultar validaciones fallidas.
- No marcar release listo si build o tests criticos fallan sin aceptacion explicita.
- Si hay deploy preview, registrar URL y fecha.
