# Docs

Documentacion viva del proyecto concreto.

## Leer Segun Necesidad

- `project-overview.md`: contexto del producto.
- `architecture/`: aplicacion local de la arquitectura base, boundaries y decisiones.
- `design-system/`: tokens, componentes, primitives y migracion visual.
- `features/`: comportamiento por feature.
- `integrations/`: APIs, envs y servicios externos.
- `user-logic/`: roles, permisos, reglas de negocio y validaciones.
- `quality/`: testing y coverage.
- `operations/`: setup, scripts y release.
- `log/`: changelog y decisiones navegables.

## Regla

`docs/` describe el proyecto concreto. Si una regla debe guiar futuros proyectos, moverla o resumirla en `wiki/`.
